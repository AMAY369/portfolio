## üí° **Pro Tips for Service Definitions**

### **1. Be Specific, Not Generic:**
‚ùå "I build websites"
‚úÖ "Modern, responsive web applications built with React and Next.js for startups launching MVPs"

### **2. Lead with Client Benefit:**
‚ùå "I use React, Node.js, and PostgreSQL"
‚úÖ "Scalable web applications that grow with your business, built with modern tech stack"

### **3. Show Expertise Through Details:**
- Specific technologies (not just "web development")
- Typical timelines (shows experience)
- Clear deliverables (no ambiguity)

### **4. Price Transparently but Flexibly:**
- Ranges (not exact prices) maintain negotiation room
- "Starting at" language sets minimum expectations
- Multiple pricing models (hourly, project, retainer) give options

### **5. Process Builds Trust:**
- Detailed process = professionalism
- Shows you've done this before
- Reduces client anxiety about working with you
